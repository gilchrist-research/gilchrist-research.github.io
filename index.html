G-2026-081 Smart Contracts

EVM Liquidity Pool Router Reentrancy

Critical

A vulnerability was discovered in the pool routing engine where token transfers triggered user-defined callbacks before correcting state balances. Under specific race conditions, this allowed multiple token withdrawals in a single block transaction.

DEXRouter.sol Auto-Patched by Agent v4.1.2 
-  uint256 balance = tokenBalances[msg.sender];
-  require(token.transfer(msg.sender, balance), "Transfer failed");
-  tokenBalances[msg.sender] = 0;
+  uint256 balance = tokenBalances[msg.sender];
+  tokenBalances[msg.sender] = 0;
+  require(token.transfer(msg.sender, balance), "Transfer failed");
